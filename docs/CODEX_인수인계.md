## 현재 방향

- `Flow Classic Plus`의 이미지 워커 운영 지식을 가져오되, Grok 전용 자동화로 새로 분리
- 워커 종류는 하나만 먼저 만든다: `Grok 이미지 워커`
- 동영상/다운로드 전용 워커는 아직 제외

## 핵심 결정

1. 프롬프트 파일 형식은 단순하게 간다.

```txt
001 : @1 cute 3d pixar boy...
|||
```

2. `@1~@5`는 업로드한 레퍼런스 이미지 1~5번을 뜻한다.
3. `@` 선택은 좌표 클릭보다 `아래키 + Enter`로 처리하는 방향이 더 안전하다.
4. 전송은 Enter가 아니라 우측 화살표 버튼 클릭으로 고정한다.
5. 레퍼런스 이미지는 세션 시작 때 먼저 업로드하고, 새로고침 후 재사용한다.

# 최근 업데이트

- 2026-04-09: `start_run()`은 이제 더 이상 예시 프롬프트만 대기열에 넣는 UI 데모가 아니라, 실제 Playwright 작업 스레드를 띄워 `레퍼런스 업로드 -> 프롬프트 입력 -> @1~@5 선택 -> 화살표 전송 -> 다운로드 저장` 1차 흐름을 실행한다.
- 2026-04-09: Grok 자동화는 현재 1차 휴리스틱 버전이다. 입력창/플러스 버튼/전송 버튼/다운로드 버튼/최신 결과 카드 탐색을 DOM+위치 점수 방식으로 잡고 있어서, 사이트 UI가 바뀌면 이 후보 탐색부터 다시 보면 된다.
- 2026-04-09: Grok Worker는 이제 `작업봇 창 열기`와 실제 실행용 Playwright context 둘 다 `chrome` 대신 `msedge` 채널을 우선 사용한다. 수동 접속은 되는데 자동화 크롬 세션만 Cloudflare 차단이 걸리는 상황을 줄이기 위한 변경이고, Edge 채널 실패 시에만 일반 Chromium fallback으로 내려간다.
