# Grok Worker 실사용 매뉴얼

이 문서는 **지금 실제로 쓰는 방식**만 적어 둔 문서입니다.

예전 문서에는 `extension/` 중심 설명이 많이 남아 있지만,  
2026-04-13 현재 사용자가 실제로 돌리는 것은 **standalone `Grok Worker` + 기존 Edge 연결 방식**입니다.

## 1. 지금 실제로 쓰는 구조

- 실제 실행 앱: `main.py` + `grok_worker/`
- 브라우저 방식: 사용자가 먼저 연 Edge에 `connect_over_cdp`로 붙음
- 워커1 Edge 포트: `9222`
- 워커2 Edge 포트: `9223`
- 워커3 Edge 포트: `9224`

중요:

- `extension/` 폴더는 참고/실험 흔적이 남아 있지만, 지금 실사용 기준 설명은 이 문서를 따른다.
- 다음 Codex는 현재 상태를 설명할 때 **확장 프로그램이 주 개발축이라고 단정하면 안 된다.**
- 더 강하게 말하면, `extension/`은 현재 **폐기 예정**으로 보고 standalone `Grok Worker`만 실사용 대상으로 취급한다.

## 3. 현재 화면 동작 기준

### 대기열 / 로그

- 기본값은 **로그 숨김**
- 기본 화면은 대기열이 최대한 넓게 보여야 정상
- `로그 보기`를 눌렀을 때만 오른쪽 로그 패널이 나타나야 함
- `로그 숨기기`를 누르면 로그 패널은 **완전히 사라져야 함**
- 이건 “거의 안 보이게”, “아주 좁게”, “패널만 남게”가 아님
- 구현 기준도 분명함:
  - 숨김 = 레이아웃에서 로그 패널 자체를 제거
  - 표시 = 다시 붙이기
  - 즉 `pack_forget()` / `pack()` 같은 완전 제거 방식이어야 함
  - PanedWindow에서 폭만 0처럼 보이게 줄이는 식의 임시 처리 금지

### Edge 창

- Edge 창은 **위치는 자유**
- 창 크기만 작업용 크기로 맞춤
- 마지막에 쓰던 위치는 설정 파일에 저장되고, 다음 실행 때 다시 그 위치로 열리도록 보강되어 있음

### 생성 후 상태 표시

- `현재 상태`는 `20초 -> 19초 -> 18초`처럼 실시간 카운트다운으로 보여야 정상
- Flow Classic Plus처럼 초가 계속 줄어들지 않으면 UI 상태 갱신이 깨진 것

### 프롬프트 파일 요약

- 프롬프트 파일 정보는 `045~080` 같은 단순 범위가 아니라 실제 들어 있는 번호 목록으로 보여야 정상
- 예: `총 5개 | 045,046,049,050,080`
- 연결 장면 `S045>S046 Prompt`도 시작 번호 `045`로 읽어야 함

## 5. 절대 자주 터지는 실수

### A. Edge 실행 파일 수정 후 실제 실행 테스트 안 함

이 프로젝트 폴더에는 공백이 들어 있음:

- `C:\Users\jaekw\Grok Worker`

그래서 아래 같은 수정은 아주 잘 깨진다.

- `start "" python ...`
- 상대경로 `edge_launcher.py`
- 인용부호 빠진 `%~dp0`
- CMD/VBS에서 공백 경로 전달

반드시 해야 하는 테스트:

1. `3_open_edge_for_grok.cmd` 직접 실행
2. `4_open_edge_for_worker2.cmd` 직접 실행
3. Edge 창이 실제로 떠야 통과

`--help`만 돌리거나 문법 검사만 하면 통과로 보면 안 된다.

### B. 로그 숨기기가 “좁히기”처럼 남는 문제

- 사용자가 원한 동작은 `숨기기 = 아예 안 보임`
- 따라서 로그 패널은 숨김 시 PanedWindow에서 완전히 빠져야 함
- 애매한 우회 구현 금지:
  - 폭 1로 줄이기
  - sash 이동만 하기
  - 배경색으로 가리기
  - 오른쪽 빈 틀만 남기기
- 다음 Codex는 이 요구를 “대충 비슷하게” 해석하면 안 됨
- **정확한 요구는 완전 제거 / 완전 복귀다**

### C. 다운로드 뒤 중앙 클릭으로 큰 이미지가 다시 눌리는 문제

- 다운로드 패널 닫는다고 화면 중앙을 클릭하면 안 됨
- Escape 기반 닫기만 유지하는 쪽이 안전

### D. 다음 번호 넘어가기 직전 보이지 않는 div를 버튼으로 오탐

- `absolute bottom-2 right-2 ...` 같은 보이지 않는 컨테이너를 클릭 후보로 잡으면 안 됨
- 모드/비율 버튼 클릭 실패는 전체 중단이 아니라 `경고 후 계속 진행`이 더 안전

## 6. 최근 실제 반영된 핵심 내용

- Edge 다운로드 버튼 탐지 보강
- 연결된 Edge의 실제 viewport 기준으로 탐색 보정
- Edge는 위치 자유, 크기만 맞춤
- 마지막 Edge 위치 저장/재사용 보강
- 다운로드 뒤 중앙 클릭 제거
- 상태 카운트다운 실시간 표시
- 로그 패널 기본 숨김 / 버튼으로만 표시
- 모드/비율 버튼 클릭 실패 시 전체 실행 중단 대신 경고 후 계속 진행
