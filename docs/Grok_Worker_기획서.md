# Grok Worker 기획서

## 1. 프로젝트 목표

`Grok Imagine` 사이트에서 이미지 생성만 자동화하는 별도 워커를 만든다.

Flow 이미지 워커에서 검증된 아래 요소를 최대한 재사용한다.

- 워커 중심 UI
- 프로젝트 선택
- 프롬프트 파일 선택
- 번호 범위 실행
- 저장 폴더 관리
- 대기열 표시
- 완료/실패 로그

대신 Grok 전용으로 바뀌는 부분은 아래다.

- `@S001` 같은 Flow 에셋 검색이 없음
- 대신 최대 5개 이미지 업로드 후 재사용
- 전송은 `Enter`가 아니라 우측 화살표 버튼 클릭
- 다운로드는 결과 화면에서 다운로드 후 파일명 지정 저장

---

## 2. 사이트 동작 기준 정리

### 기본 프롬프트 창

- 하단 입력창이 기본 상태
- 좌측 `+` 버튼을 누르면 이미지 추가 UI가 열림

### 이미지 업로드 단계

1. `+` 클릭
2. `Upload or drop images` 클릭
3. 파일 열기 창에서 미리 정해둔 폴더의 이미지 선택
4. 최대 5장까지 업로드

### 업로드 후 재사용 단계

- 업로드 직후에는 첨부된 상태 UI가 보임
- 새로고침 후 다시 기본 입력창으로 돌아감
- 다시 `+`를 누르면 방금 업로드한 이미지들이 위쪽 리스트에 뜸
- 여기서 1~5개 이미지를 바로 클릭해 첨부 가능

### `@` 선택 단계

- 프롬프트 입력 중 `@`를 누르면 이미지 선택 팝업이 뜸
- 팝업에는 업로드된 이미지가 `Image 1`, `Image 2` 식으로 뜸
- 이미지 개수에 따라 팝업 높이가 달라질 수 있음
- 좌표 클릭보다 `위/아래키 + Enter`가 더 안정적

### 제출

- 우측 끝 화살표 버튼 클릭으로만 전송

### 결과 다운로드

1. 결과 이미지 열기
2. 다운로드 버튼 클릭
3. 파일명 변경
4. 저장 폴더에 저장
5. 다시 초기 화면으로 복귀

## 3. MVP 범위

1차 버전은 아래만 만든다.

- `Grok 이미지 워커` 1종
- 프로젝트 URL 저장
- 프롬프트 파일 선택
- 번호 범위 / 개별 번호 실행
- 레퍼런스 이미지 폴더 선택
- 저장 폴더 선택
- 최대 5개 업로드 이미지 세션 준비
- `@1 ~ @5` 자동 선택
- 생성 후 다운로드
- 파일명 저장
- 대기열 / 성공 / 실패 표시

제외:

- 동영상
- 여러 워커 종류
- 복잡한 이어달리기
- 다운로드만 따로 하는 별도 워커

## 4. 추천 프롬프트 파일 형식

가장 단순한 형식으로 통일한다.

```txt
001 : @1 cute 3d pixar style boy, cinematic lighting
|||

002 : @1 @2 same character, shocked expression, warm top light
|||
```

규칙:

- `001`은 작업 번호
- `:` 뒤는 실제 Grok 입력창에 들어갈 프롬프트 본문
- `|||`는 블록 끝
- `@1 ~ @5`는 업로드된 레퍼런스 이미지 선택 번호

### 실제 입력 흐름

예시:

```txt
002 : @1 @2 same character, shocked expression, warm top light
|||
```

실행 시:

1. 입력창에 `same character, shocked expression, warm top light` 기본 텍스트를 조합
2. `@1`을 만나면 `@` 입력
3. 팝업에서 1번 이미지 선택
4. `@2`를 만나면 다시 `@`
5. 팝업에서 2번 이미지 선택
6. 나머지 본문 입력
7. 우측 화살표 제출

즉 `@1`, `@2`는 텍스트 그대로 보내지지 않고, 실제 이미지 첨부 동작으로 바뀐다.

## 6. 실행 순서 설계

### 세션 준비 단계

1. Grok Imagine 프로젝트 페이지 열기
2. 입력창 확인
3. `+` 클릭
4. 업로드할 레퍼런스 이미지 1~5장 업로드
5. 페이지 새로고침
6. 다시 입력창 기준 화면 복귀

### 각 번호 작업 단계

1. 번호별 프롬프트 읽기
2. 본문 안의 `@1~@5` 토큰 파싱
3. 입력창 초기화
4. 본문을 순서대로 입력
5. `@n` 나오면:
   - `@` 입력
   - 아래 방향키 `n-1`번
   - Enter
6. 프롬프트 완성 후 화살표 버튼 클릭
7. 결과 생성 대기
8. 결과 이미지 열기
9. 다운로드
10. 파일명 `G001`, `G002`처럼 저장
11. 저장 확인 후 성공 처리
12. 초기 화면 복귀

## 7. 안정성 포인트

### A. 좌표 클릭 최소화

가장 중요한 원칙:

- `@` 선택은 좌표보다 키보드 선택을 우선
- 정해진 위치 클릭보다 `입력창/버튼 후보 탐색` 우선

### B. 업로드 이미지 순서 고정

업로드 순서가 바뀌면 `@1~@5` 의미가 바뀐다.

그래서 세션 시작 때:

- 파일명을 정렬해 업로드 순서를 고정
- 업로드 완료 후 썸네일 순서를 기록
- 필요 시 사용자에게 `1번=어떤 이미지` 요약 표시

### C. 다운로드명 충돌 방지

- 기본 파일명: `G001.png`
- 이미 있으면 `G001 (2).png` 대신
  - 덮어쓰기 여부 선택
  - 또는 타임스탬프 접미사 옵션

### D. 실패 처리

실패 기준:

- 업로드 팝업 미탐지
- `@` 이미지 선택창 미탐지
- 제출 버튼 미탐지
- 결과 생성 타임아웃
- 다운로드 실패

실패 시:

- 즉시 다음 번호 이동
- 대기열 붉은색 표시
- 실패 번호 복붙 지원
