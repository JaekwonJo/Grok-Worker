# Grok Worker 기획서

## 1. 프로젝트 목표

`Grok Imagine` 사이트에서 이미지 생성만 자동화하는 **브라우저 확장 프로그램**을 만든다.

처음에는 standalone Playwright 워커를 시도했지만, 실제 테스트에서 x.ai / Cloudflare가 자동화 브라우저를 입구에서 차단했다. 그래서 지금은 방향을 바꿨다.

- 예전 standalone `Grok Worker`는 보관
- 실제 제품은 `manifest v3` 확장 프로그램으로 진행

재사용할 핵심 아이디어는 아래다.

- 번호 기반 프롬프트 파일
- 대기열/실패 번호 관리
- 사람형 입력 속도
- 저장 폴더명 규칙

Grok 전용으로 바뀌는 부분은 아래다.

- 별도 브라우저를 띄우지 않음
- 사용자의 실제 Edge/Chrome 탭 안에서 동작
- 레퍼런스 이미지는 확장 프로그램 라이브러리에 저장
- 프롬프트의 `@S999`, `@S998`를 보고 같은 이름 이미지를 자동 첨부
- 전송은 우측 화살표 버튼 클릭
- 다운로드는 페이지 안의 다운로드 버튼을 눌러 진행

---

## 2. 사이트 동작 기준 정리

### 기본 프롬프트 창

- 하단 입력창이 기본 상태
- 좌측 `+` 버튼을 누르면 이미지 추가 UI가 열림

### 이미지 첨부 단계

1. 사용자가 확장 프로그램 사이드패널에 레퍼런스 이미지들을 미리 저장
2. 각 이미지에 별칭 부여
   - 예: `S999`, `S998`, `S997`
3. 실행 시 확장 프로그램이 현재 프롬프트에서 필요한 별칭만 골라
4. Grok 입력창 옆 `+` / 파일 입력칸을 이용해 현재 프롬프트에 첨부

### 제출

- 우측 끝 화살표 버튼 클릭으로만 전송

### 결과 다운로드

1. 결과 이미지 열기
2. 다운로드 버튼 클릭
3. 파일명 변경
4. 저장 폴더에 저장
5. 다시 초기 화면으로 복귀

## 3. MVP 범위

1차 버전은 아래만 만든다.

- 확장 프로그램 사이드패널 1종
- 프롬프트 텍스트 입력 / txt 불러오기
- 번호 범위 / 개별 번호 실행
- 레퍼런스 이미지 라이브러리 저장
- `@S999`, `@S998` 별칭 자동 매칭
- 현재 프롬프트에 필요한 이미지 자동 첨부
- 생성 후 다운로드 버튼 클릭
- 파일명 규칙 지정
- 대기열 / 성공 / 실패 표시

제외:

- 동영상
- 여러 워커 종류
- 복잡한 이어달리기
- 다운로드만 따로 하는 별도 워커

## 4. 추천 프롬프트 파일 형식

가장 단순한 형식으로 통일한다.

```txt
001 : @S999 cute 3d pixar style boy, cinematic lighting
|||

002 : @S999 @S998 same character, shocked expression, warm top light
|||
```

규칙:

- `001`은 작업 번호
- `:` 뒤는 실제 Grok 입력창에 들어갈 프롬프트 본문
- `|||`는 블록 끝
- `@S999`, `@S998`는 확장 프로그램 라이브러리의 이미지 이름

### 실제 입력 흐름

예시:

```txt
002 : @S999 @S998 same character, shocked expression, warm top light
|||
```

실행 시:

1. 확장 프로그램이 `S999`, `S998` 이미지를 라이브러리에서 찾음
2. 해당 이미지들을 현재 Grok 프롬프트에 첨부
3. 입력창에는 `S999 S998 same character...` 형태로 `@`를 뺀 본문을 입력
4. 우측 화살표 제출

즉 `@S999`, `@S998`는 텍스트 그대로 보내지지 않고, 실제 이미지 첨부 지시로 쓰인다.

## 5. 추천 내부 구조

```txt
Grok Worker/
├─ README.md
├─ docs/
│  ├─ Grok_Worker_기획서.md
│  └─ CODEX_인수인계.md
├─ samples/
│  └─ grok_prompt_example.txt
├─ extension/
│  ├─ manifest.json
│  ├─ background.js
│  ├─ content.js
│  ├─ sidepanel.html
│  ├─ sidepanel.css
│  ├─ sidepanel.js
│  └─ lib/
│     ├─ prompt-parser.js
│     └─ ref-db.js
└─ (보관용 standalone 파일들)
```

### 모듈 역할

- `manifest.json`
  - 확장 프로그램 권한 / content script / side panel 정의

- `background.js`
  - 실행 세션 관리
  - 다운로드 파일명 제안
  - sidepanel <-> content script 연결

- `content.js`
  - 현재 Grok 탭 DOM 제어
  - 이미지 첨부 / 프롬프트 입력 / 전송 / 다운로드

- `sidepanel.js`
  - 설정 저장
  - 프롬프트 입력
  - 레퍼런스 이미지 라이브러리 UI
  - 대기열 표시

- `lib/prompt-parser.js`
  - `001 : 본문 |||` 파싱
  - `@S999` 토큰 분리

- `lib/ref-db.js`
  - 레퍼런스 이미지 blob / 별칭 저장

## 6. 실행 순서 설계

### 각 번호 작업 단계

1. 번호별 프롬프트 읽기
2. 본문 안의 `@S999`, `@S998` 토큰 파싱
3. 입력창 초기화
4. 레퍼런스 라이브러리에서 필요한 이미지 찾기
5. 현재 프롬프트에 이미지 첨부
6. `@`를 뺀 본문을 `S001 Prompt : ...` 형태로 입력
7. 화살표 버튼 클릭
8. 결과 생성 대기
9. 결과 이미지 열기
10. 다운로드
11. 파일명 `S001.png`, `S002.png`처럼 저장
12. 저장 확인 후 성공 처리

## 7. 안정성 포인트

### A. 별도 자동화 브라우저 금지

가장 중요한 원칙:

- Playwright standalone 브라우저를 메인 방식으로 쓰지 않음
- 사용자가 실제로 쓰는 Edge/Chrome 탭 안에서만 동작

### B. 좌표 클릭 최소화

가장 중요한 원칙:

- 정해진 위치 좌표보다 DOM 후보 탐색 우선
- `+`, 입력창, 화살표, 다운로드 버튼은 위치 점수 + 텍스트로 탐색

### C. 이미지 이름 매칭 안정화

순서 기반 `@1~@5`보다, 이름 기반 `@S999`가 더 안전하다.

그래서:

- 레퍼런스 이미지는 이름으로 저장
- 프롬프트에서도 같은 이름으로 호출
- 순서가 바뀌어도 의미가 안 깨짐

### D. 다운로드명 충돌 방지

- 기본 파일명: `G001.png`
- 이미 있으면 `G001 (2).png` 대신
  - 덮어쓰기 여부 선택
  - 또는 타임스탬프 접미사 옵션

### E. 실패 처리

실패 기준:

- 업로드 팝업 미탐지
- `@` 이미지 선택창 미탐지
- 제출 버튼 미탐지
- 결과 생성 타임아웃
- 다운로드 실패

실패 시:

- 즉시 다음 번호 이동
- 대기열 붉은색 표시
- 실패 번호 복붙 지원
